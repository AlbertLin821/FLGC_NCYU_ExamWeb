作為專業網站開發設計師，針對您的線上考試平台需求（國立中興大學英文前後測驗），我建議先制定詳細開發計畫，涵蓋每個畫面、功能、安全機制及成本優化，以確保平台穩定、防弊且低成本。

計畫需考慮防作弊核心技術，如全螢幕鎖定與視窗監控，這是避免學生跳窗作弊的關鍵實作方式。

## 開發架構概述

平台採用全端分離架構：前端React + TypeScript（防弊監控）、後端FastAPI或Node.js（API服務）、資料庫PostgreSQL（低成本RDS）。
部署使用Docker容器化，AWS或台灣雲端（如中華電信雲）主機，支援1900學生同時考試需預估峰值負載（每人5題，AI評分）。
安全使用JWT token驗證，HTTPS全程加密，老師端mail邀請經後端驗證email服務（如AWS SES）。

## 學生端畫面流程

### 首頁選擇畫面

顯示「我是老師 / 我是學生」按鈕，學生點擊後輸入學號姓名

### 學生登入畫面

輸入學號+姓名，後端即時查DB驗證（避免重複考試）。驗證邏輯：SQL查詢`SELECT * FROM students WHERE student_id=? AND name=? AND exam_status='pending'`。

### 考卷選擇畫面

顯示「目前該學生需考的考卷」（依班級過濾，如英文前測/後測，不同考場不同難度）。列表：考卷名稱、難度、剩餘時間。點擊進入考場，記錄開始時間。

### 考試主畫面（單一題目）

每題顯示兩個英文單字，要求造句（textarea輸入），提交後鎖定下一題（防跳題）。全螢幕鎖定：使用Fullscreen API `document.documentElement.requestFullscreen()`，監控`visibilitychange`事件偵測跳窗，觸發警告或結束考試。
進度條顯示5/5題，計時器倒數，無法後退。防弊額外：禁用右鍵/複製（`contextmenu`、`copy`事件preventDefault），鍵盤監控Alt+Tab。

## 老師/管理端畫面流程

### 登入畫面

帳號密碼登入（bcrypt雜湊），管理者可mail邀請新老師（後端發送驗證連結，點擊後設角色）。畫面：email/密碼欄位，忘記密碼重置。

### 學生管理畫面

Excel匯入（學號、姓名、班級）：前端FileReader解析CSV，後端批量INSERT SQL `INSERT INTO students (id, name, class_id) VALUES ... ON DUPLICATE KEY UPDATE`。
列表顯示所有學生，篩選班級，匯出成績Excel。

### 班級管理畫面

新增/編輯班級，邀請老師（共享權限：查詢特定class_id）。表格顯示班級、學生數、分數統計（平均/最高/分布）。

### 題目管理畫面

CRUD題目：新增考卷、題型（造句：兩個單字JSON），指定班級發放。儲存DB表`questions (id, words1, words2, difficulty, exam_id)`。Excel匯入題庫支援。

### 成績後台畫面

表格顯示學生分數、答題句子、AI評分詳情（正確性0-100%）。篩選班級/學生，匯出報告。分數計算：存DB後計算平均。[^8]

## AI造句評分實作

每題造句送OpenAI GPT-5 mini API（低成本），prompt engineering確保準確：「評分以下句子是否正確使用單字'{word1}'和'{word2}'，文法正確、意思通順。給0-100分，並解釋原因。只回分數和簡短說明。」
Token估算：輸入~100 tokens（prompt+句子），輸出~50 tokens。一題成本~0.00015 USD（GPT-5 mini \$0.25/M輸入+\$2/M輸出）。1900人x5題=9500請求，總~1.4 USD。
優化：批次API（24小時內50%折扣），快取常見錯誤；存答案DB提升追蹤（非只分數）。正確性：測試多prompt迭代，準確率>90%。

## 防作弊技術細節

全螢幕強制+ Page Visibility API偵測隱藏（`document.hidden` true時警告）。
進階：WebRTC鏡頭眼動追蹤（face-api.js），記錄異常log上傳後端。禁用開發工具（偵測F12）。無需第三方如ProctorU，純前端+後端log。
測試：模擬跳窗，確保自動結束考試並標記，並記錄下學生異常行為於老師端跳出通知。
